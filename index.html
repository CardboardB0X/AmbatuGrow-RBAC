1. What is an Enterprise Resource Planning (ERP) System?

An ERP is a centralized software system designed to coordinate and manage all the core operational departments of a business—ranging from purchasing, inventory management, warehousing, e-commerce, customer relations, human resources, to finance.

By feeding data into a single master database, an ERP eliminates "data silos" (where one department is unaware of the logs in another). For example, as soon as a customer completes checkout in E-Commerce:

  • The Inventory module immediately decrements stock and updates the online channel.
  • The CRM generates a sales order and updates their transaction history.
  • The Finance module generates accounts receivable entries and updates the General Ledger.

2. Key ERP System Boundaries

ERP systems partition permissions and operations into strict boundaries to prevent critical security errors:

  • Supply Chain vs. Finance: While operators manage stock transactions, only accounting personnel can modify financial records or approve invoice payouts.
  • Purchasing vs. Approvals: Employees request goods via a Requisition, but a separate manager role must approve it before a Purchase Order is dispatched to a supplier. This dual-authorization prevents unauthorized expenditures.

Procurement & Supplier Inbound Lifecycle

The procurement pipeline coordinates material acquisitions:

  • PR Initiation: The department submits a Purchase Requisition.
  • Approval Gate: The Department Manager verifies budgets and signs off.
  • PO Dispatched: A formal Purchase Order (PO) detailing products and unit costs is generated and dispatched to the supplier.
  • Dock Stock-in: Upon delivery, Warehouse Operators verify the SKUs, log quantities in the target zone, and write a 'Stock-in' log.
  • 3-Way Match Verification: The Accountant receives the Supplier Invoice and verifies it against the PO and the Dock Receipt. If matching, payment is released and logged in the General Ledger.

Sales Fulfillment & Customer Outbound Lifecycle

The customer fulfillment loop ensures rapid dispatch and billing integrity:

  • CRM Quote: Sales Reps query the customer profile, apply discount matrices, and issue a Quotation.
  • Order Creation: Once confirmed, a Sales Order is created with payment terms.
  • Stock Check: The system queries the inventory levels. If stock is available, it routes the order to the warehouse. If out of stock, the order is flagged 'On Hold' and triggers a Procurement request.
  • WMS Stock-out: Operators retrieve the items, package them, decrement inventory locations, and log a 'Stock-out' transaction.
  • Invoicing & Receivable: The finance team creates a customer invoice. When payment arrives, accounts receivable is reconciled, and the General Ledger is updated.

AmbatuGrow uses a highly normalized relational database layout to guarantee transactional integrity (ACID compliance) and prevent data duplication.

Deploying to Vercel in Seconds

Because this interactive portal is a static single-page application built on standard HTML, CSS, and Vanilla JavaScript, it requires zero server-side builds or complex configurations. You can host it globally on Vercel for free using the following deployment guidelines.

1

Verify the Code on GitHub

Ensure that all changes have been pushed to the remote repository. Since Vercel integrates directly with GitHub, it will watch your branches and compile the page on every single commit.

Repository URL HTTPS 
https://github.com/CardboardB0X/AmbatuGrow-RBAC.git
2

Sign In to Vercel and Import

Navigate to vercel.com, sign in using your GitHub account, and click the Add New Project button. Search for the AmbatuGrow-RBAC repository and click Import.

3

Configure Settings & Deploy

Since the project is a static site with index.html in the root directory, Vercel will auto-detect the configuration. Leave all settings at their default values and click Deploy. Your site will be live on a custom .vercel.app domain within 15 seconds!
